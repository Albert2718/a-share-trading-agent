# 产品需求

## 产品目标

构建一个面向个人投资者的 A 股投研控制台：用户以自然语言描述目标，唯一的 LangGraph Agent 调用受控工具完成行情查询、持仓管理、结构化 Memory、用户文档检索和后台深度研究，同时保留必要的结构化表单入口。

## 核心场景

1. 行情查询：按股票代码查询实时或最近交易日价格、涨跌幅、成交量和历史走势。
2. 持仓管理：读取持仓；新增或修改持仓必须先展示确认操作，确认后才写入 SQLite。
3. 深度研究：自然语言或表单提交研究任务，由 Celery 后台执行并展示进度与报告。
4. 长期记忆：保存明确的风险偏好、投资风格、约束和关注列表，写入前必须确认。
5. 用户知识库：上传 PDF、Markdown 或 TXT，经 chunk、Embedding 和 Qdrant 索引后供 Agent 检索并返回来源。

## 交互要求

- 登录后的聊天区是主界面，研究、持仓和任务收纳在工具侧栏。
- 用户消息发送后应立即显示，Agent 回答后续升级为 SSE 流式输出。
- Markdown 必须安全渲染；行情、持仓、任务和报告逐步使用结构化结果卡。
- 桌面端不产生页面级滚动，聊天和各详情工作区独立滚动；深度研究、持仓和任务均由左侧导航进入。
- 视觉采用深色蓝黑投研控制台风格，中文字体使用霞鹜文楷。

## 安全与产品边界

- 项目提供研究辅助，不承诺收益，也不执行自动交易。
- 浏览器不能直接访问 LLM Key、数据库、Redis、Qdrant 或外部数据源密钥。
- 所有用户数据查询必须按 `user_id` 隔离。
- 持仓和 Memory 属于确认写入；研究和文档索引属于可靠后台任务。
- LLM、行情源或单个研究角色不可用时应明确降级，不得编造数据。
