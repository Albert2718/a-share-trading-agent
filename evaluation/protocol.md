# Agent 现实性评测协议

## 1. 目标与期限

本评测验证 Agent 在不使用未来信息的前提下，对固定 A 股股票池作出可复核预测的能力。评测分为两条相互独立的轨道：

1. **次日预测评测**：每个交易日收盘后预测下一交易日的涨跌方向、收盘价和价格区间。
2. **阶段趋势评测**：以 2026-07-14 收盘后可获得的信息为起点，预测 2026-07-23 收盘时的阶段趋势和价格；2026-07-23 收盘后结算，供 2026-07-24 答辩使用。

阶段趋势评测只有 7 个交易日，结论必须称为“阶段趋势”或“波段判断”，不得表述为已经证明长期投资能力。公司长期发展趋势可以作为研究依据和定性结论，但不与短期价格命中率混为一谈。

## 2. 固定股票池

股票池由系统在首次运行时从当期沪深 300 成分股中选出 20 只，并冻结至答辩结束。筛选规则如下：

1. 排除名称包含 ST 或退市标记的证券。
2. 排除上市不足两年、近 250 个交易日行情不完整、当前停牌或无法取得有效收盘价的证券。
3. 按近 60 个交易日平均成交额衡量流动性。
4. 采用行业分层后的贪心选择：优先选择流动性较高且能增加行业覆盖的证券；同一一级行业最多选择 2 只，直到选满 20 只。
5. 将成分股来源日期、筛选时间、行业、流动性指标、入选原因和规则版本写入 `evaluation/stock_pool.json`。

股票池不得因预测结果不佳而更换。只有退市、长期停牌、被实施 ST 或持续无法获得有效数据时才允许替换。替换必须使用同一筛选规则，并记录原股票、替代股票、原因和时间；历史记录继续保留原股票。

## 3. 数据截止与防数据穿越

每条预测必须记录 `generated_at`、`as_of_trade_date` 和 `target_trade_date`。生成预测前必须验证：

- 当日市场已经收盘；
- 行情源的最新完整交易日等于 `as_of_trade_date`；
- 新闻、公告和研究证据的发布时间不晚于 `generated_at`；
- 下一交易日由交易日历计算，不按自然日加一天；
- 缓存数据保留原始获取时间和来源时间。

任何一项无法验证时，不得静默使用旧数据或推测数据。程序应记录失败原因，并要求重新运行。预测生成后发现的新闻不得补入已有预测。

价格预测和结算统一使用不复权收盘价，以便与市场显示价格直接比较。LSTM 可使用前复权序列生成收益率，但该收益率必须应用到预测时的不复权收盘价。除权除息样本单独标记，并在总指标之外提供剔除该类样本后的指标。

## 4. 研究与预测流程

完整 Agent 使用现有深度研究编排器收集量化、基本面、新闻、市场情绪和风险证据。各分析模块先输出结构化证据、方向、强度和时效，再由 CIO 综合形成预测。

LSTM 仅作为量化证据之一，不得覆盖新闻、公告、基本面或风险证据。系统同时保存 LSTM 的独立预测，作为对照结果。最终 Agent 预测必须满足以下约束：

- 给出看涨或看跌二分类结论；
- 给出预计涨跌幅、预计收盘价、价格区间和置信度；
- 每项关键判断关联实际工具结果及其时间；
- 预测幅度受近期实现波动率约束；超出正常波动范围时必须存在明确、可追溯的强催化证据；
- 数据不足时允许失败或降低置信度，不允许编造替代数据。

## 5. 预测记录

次日预测至少保存以下字段：

- 预测 ID、规则版本、生成时间、截止交易日和目标交易日；
- 股票代码、名称、行业和预测时的不复权收盘价；
- Agent 的方向、预计涨跌幅、预计收盘价、区间下限、区间上限和置信度；
- LSTM 的方向、预计涨跌幅、预计收盘价和置信度；
- 深度研究的主要证据、来源、发布时间和各分析模块结论；
- 模型、提供商和 LSTM 检查点标识；
- 数据缺失、来源失败、降级和其他警告。

阶段趋势记录还必须保存：

- 固定目标日 2026-07-23；
- 公司与行业发展趋势；
- 阶段方向、目标价格、价格区间和置信度；
- 核心逻辑、预期催化因素和主要风险。

每日记录写入 `evaluation/data/`，累计报告写入 `evaluation/reports/`。这两个目录仅保存在本地并加入 `.gitignore`；评测代码、协议和冻结后的股票池配置可以纳入版本控制。

历史预测采用追加写入。若同一预测 ID 已存在，程序必须拒绝覆盖。每个记录文件保存自身内容哈希和前一记录哈希，形成连续校验链；重新运行只补充缺失股票或执行尚未完成的结算。

## 6. 结算规则

对股票在交易日 T 收盘后生成 T+1 预测：

- 若 T+1 实际收盘价高于 T 实际收盘价，实际方向为上涨；
- 若 T+1 实际收盘价低于 T 实际收盘价，实际方向为下跌；
- 若两者相等，实际方向为平盘。平盘样本保留价格误差，但不计入二分类方向命中率。

Agent 或 LSTM 的预测方向与实际上涨或下跌一致时记为方向命中。实际收盘价落入预测区间时记为区间覆盖。绝对百分比误差不超过 1% 时记为价格容差命中。

2026-07-23 收盘后，阶段趋势评测使用当日不复权收盘价结算，并生成核心逻辑、催化因素和风险因素的证据对照表。定性结论保留证据和人工复核结果，不强行转换为数值分数。

## 7. 评价指标

次日评测分别统计完整 Agent 和 LSTM，不设置简单价格基线。指标包括：

- 涨跌方向命中率；
- 收盘价 MAE、RMSE 和 MAPE；
- 绝对百分比误差不超过 1% 的比例；
- 预测区间覆盖率；
- 单次最大绝对误差和最大绝对百分比误差；
- 预测覆盖率，即成功生成预测的股票数除以固定股票池数量；
- 最近 5 日和最近 20 日滚动指标；
- 按股票、行业和置信度区间分组的指标。

本评测不模拟买入、卖出、持仓或资金曲线，因此不计算收益率和最大回撤。

## 8. 运行与容错

统一手动入口为：

```powershell
python run_agent.py evaluate daily
```

首次有效运行负责冻结股票池、生成阶段趋势预测，并生成下一交易日预测。后续运行先结算到期预测，再更新统计报告，最后生成下一交易日预测。

20 只股票采用受限并发执行，并在每只股票完成后写入检查点。单只股票失败不得阻塞其他股票；有效预测少于 18 只时，当日批次必须标记为“不完整”。已生成的有效预测仍进入样本级汇总，但该批次不得冒充完整的 20 股日均结果，报告必须同时展示预测覆盖率。修复数据源后可重新运行以补齐缺失股票，但不得改写已经成功生成的预测。

所有错误必须包含股票代码、阶段、数据源、时间和可执行的失败原因。日志和报告不得包含 `.env` 中的 API Key 或其他密钥。

## 9. 测试范围

`tests/evaluation/` 覆盖：

- 股票池筛选、行业上限、冻结和异常替换；
- 交易日推算、数据新鲜度和新闻时间截止；
- 涨跌方向结算、平盘处理和除权除息标记；
- MAE、RMSE、MAPE、1% 容差、区间覆盖和滚动指标；
- 历史防覆盖、哈希链和断点续跑；
- 使用模拟工具结果测试深度研究与 LSTM 的结构化集成。

真实行情、新闻、LLM 和 LSTM 的端到端调用作为手动验收测试，不进入默认单元测试，避免网络波动和 API 成本导致常规测试不稳定。
